# Отчет о качестве данных

## 1. Введение

Данный датасет содержит информацию о посещениях фитнес-центра за период с января по декабрь 2024 года. В нём представлены данные о клиентах и их тренировках, такие как возраст, пол, тип членства, виды тренировок, продолжительность, потраченные калории и статус посещения. 
Заказчиком анализа может выступать бизнес-менеджмент фитнес-клуба или команда продукта

#### Сегментация клиентов
- Выявление устойчивых групп посетителей на основе паттернов поведения: регулярности посещений, предпочитаемых видов тренировок, времени прихода, сжигаемых калорий и др.

#### Поиск аномалий и анализ причин пропусков
- Обнаружение нестандартных случаев в данных: отрицательные значения калорий, крайне низкая/высокая продолжительность тренировок при значительном расходе калорий и др.

#### Описательная аналитика и поиск закономерностей
- Профилирование групп по типам членства, возрасту и полу.

## 2. Паспорт датасета: 

В датасете 2600 строк, 10 столбцов  
Признаки:  
member_id                - Уникальный идентификатор члена фитнес-клуба<br>
visit_date               - Дата посещения<br>
age                      - Возраст посетителя в годах<br>
gender                   - Пол посетителя<br>
membership_type          - Тип членства<br>
workout_type             - Тип тренировки<br>
workout_duration_minutes - Длительность тренировки в минутах<br>
calories_burned          - Количество сожжённых калорий<br>
check_in_time            - Время регистрации входа<br>
attendance_status        - Статус посещения<br>

## 3. Основные проблемы качества данных:  

| № | Тип проблемы                 | Описание                                                                                                                       | Потенциальный риск для анализа                                   | Предлагаемое решение/ стратегия                                                                 |
|---|------------------------------|--------------------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------|-------------------------------------------------------------------------------------------------|
| 1 | Пропущенные значения         | В столбце `gender` отсутствует ~33% значений.                                                                                  | Снижение точности бизнес-решений, ухудшение качества анализа     | Удаление записей с пропусками, либо востановление пола по паттернам                             |
| 2 | Неправильные типы данных     | Строки вместо дат/времени/категорий (`visit_date`,`check_in_time`, `gender`, `membership_type`, `workout_type`).               | Невозможность временного анализа и группировки                   | Преобразование типов (datetime, category)                                                       |
| 3 | Выбросы в числовых признаках | Невозможные значения (нулевое время тренировки, количество сожженных калорий более 3000)                                       | Искажение статистических показателей (среднее, стандартное отклонение). | Фильтрация по логическим границам.                                                              |
| 4 | Неполные данные              | id не указывает на реальный идентификатор клиента в базе, а лишь отражает номер сессии в выборке                               | Нельзя провести анализ для конкретного клиента                   | Добавление поля с уникальным id для каждого клиента                                             |
| 5 | Неверность данных            | Потраченные калории в условиях одинакового вида тренировки расчитаны неверно. Посчитаны без веса/роста или по неверной формуле | Неверные рейтинги эффективности тренировок, ошибки при анализе коррелирующих параметров         | Выявление причины неправильных расчетов, пересчёт по реальным формулам, исправление и валидация |

## 4. План улучшения качества:

**Высокий приоритет:**
- Исправить критические проблемы структуры данных: отсутствие реальной идентификации клиентов и неправильные типы данных и неверный расчет калорий.

**Средний приоритет:**
- Заполнить пропуски в поле gender по паттернам тренировок и устранить выбросы в числовых данных.
- Создать автоматические проверки валидации.

**Низкий приоритет:**
- Добавить дополнительные признаки для увеличения возможностей анализа

## 5. Краткое резюме

В процессе работы научился описывать датасет фитнес-клуба, выполнять первичный обзор данных, строить графики для визуализации, выявлять структурные ошибки данных и предлагать технические решения.

Особого внимания при ИАД требуют отсутствие реальной идентификации клиентов, неверный унифицированный расчет калорий без учета типа тренировки, 33% пропусков в поле gender и преобразование типов данных.
